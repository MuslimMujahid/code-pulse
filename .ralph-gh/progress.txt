## Codebase Patterns
- Package manager: pnpm (use pnpm for all installs/runs)
- Test runner: Vitest with jsdom environment; passWithNoTests: true to exit 0 with no tests
- TypeScript: strict mode, moduleResolution: bundler, paths alias @/* → ./src/*
- CSS: Tailwind v4 via @import "tailwindcss" in globals.css (no tailwind.config.js needed)
- Next.js: App Router, 15.x, src/app directory structure
- ESLint: flat config (eslint.config.mjs), requires @eslint/eslintrc package (not bundled with next)
- Quality checks: pnpm typecheck (tsc --noEmit), pnpm test (vitest run), pnpm lint (next lint)
- Env vars: GEMINI_API_KEY, GEMINI_MODEL, CODEPULSE_MAX_COMMITS documented in .env.example
- .gitignore must include .next/, tsconfig.tsbuildinfo, .ralph-gh/agent.lock, .ralph-gh/agent.log
- simple-git: simpleGit(path) throws synchronously when the directory doesn't exist — wrap instantiation in try/catch, not just checkIsRepo()
- simple-git: use git.raw(["log", "--format=MARKER...", "--numstat"]) for reliable per-commit file stat parsing; split output on the MARKER prefix
- Shared types live in src/lib/types.ts (CommitEntry, RepoData, ParseRepoResult, GraphNode, GraphEdge, GraphData etc.)
- GraphBuilder: buildGraph() is a pure function; coChangeKey helper is duplicated locally (not re-imported from git-parser) to avoid cross-module coupling
- GraphBuilder: lastCoChangeDate is computed by scanning commits oldest-to-newest and overwriting the map entry, so the last write is the most recent date
- GraphBuilder: fileToCommits entries are ordered oldest-first (matches git-parser's reversed output), so hashes[0] gives firstCommitDate directly
- CommitSummarizer: AI_UNAVAILABLE is a string sentinel const; server-side only; uses in-memory Map cache keyed by filePath::lastCommitHash; CHUNK_SIZE=50 commits per prompt chunk; consolidation pass when >1 chunk

## 2026-04-05 - US-007: Implement POST /api/summarize route
- What was implemented: Created src/app/api/summarize/route.ts — a Next.js App Router POST handler that accepts { filePath, commits }, calls CommitSummarizer.summarize(), and streams the result back as plain text. Returns HTTP 503 { error: 'AI_UNAVAILABLE' } when GEMINI_API_KEY is missing. Returns HTTP 400 for missing/invalid parameters. Returns HTTP 500 for unexpected Gemini errors.
- Files changed: src/app/api/summarize/route.ts (new)
- **Learnings for future iterations:**
  - Streaming a pre-resolved string via ReadableStream: create a ReadableStream with a start() controller that enqueues the encoded result and immediately closes. This satisfies the "no buffering" requirement even when CommitSummarizer internally awaits the full Gemini response before returning.
  - Return `new Response(stream, { headers: { "Content-Type": "text/plain; charset=utf-8" } })` — NOT NextResponse.json() — for streaming text bodies.
  - AI_UNAVAILABLE check: compare result === AI_UNAVAILABLE (string sentinel), then return 503; all other errors from summarize() are caught and returned as 500.
  - The route is thin by design — all business logic (caching, chunking, API key handling) stays in CommitSummarizer; the route only handles HTTP concerns.
